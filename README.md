## 🧠 3. Engenharia de Prompts & “Cicatrizes”

### 🔍 Problema Inicial

Currículos qualificados sendo rejeitados sem motivo aparente.

### ⚙️ Descoberta Técnica

O problema não era falta de habilidade, mas sim:

> **Double Parsing** (dupla interpretação de dados)

Isso ocorre principalmente no **Easy Apply**, onde:

- Datas são achatadas  
- Meses de experiência são perdidos  
- Informações ficam inconsistentes  

---

### 🎯 Prompt Estratégico

```text
"Como otimizar o perfil para ser encontrado pela IA?"
```

### 🧩 Refinamento

Evolução para o conceito de:

**Selective Signal Architecture**

Ou seja:

- Sinalizar valor para recrutadores externos  
- Evitar exposição desnecessária para o empregador atual  

---

## 🚀 4. Miniguia de Estudo (Entrega Final)

### 📈 Otimização SEO no LinkedIn

O título deve seguir a estrutura:

```text
Cargo + Stack + Impacto
```
- Limite: 220 caracteres
- Objetivo: Melhor ranqueamento no LinkedIn Recruiter


### 🤖 Triagem por IA

- Score: **1 a 5**  
- Método valorizado: **STAR**
  - Situação  
  - Tarefa  
  - Ação  
  - Resultado  

🚫 Penalizações:

- Uso de “palavras de preenchimento” (ex: *hum*, *tipo*)  

---

### 🔒 Privacidade

Recomendação:

- Desativar: **"As pessoas também viram"**  
- Motivo: evitar exposição de concorrentes no seu perfil  

---

## 📖 5. Glossário de Conceitos

**Parsing**  
→ Extração automática de dados por IA  

**Double-Agent Tension**  
→ Conflito entre visibilidade no mercado vs. invisibilidade no emprego atual  

**Skills-Based Hiring**  
→ Foco no que você sabe fazer, não no seu cargo  

**Creator Mode**  
→ Libera hashtags e analytics para alcance orgânico
